The Problem

What's broken

When an LLM answers a question using web search, you have no idea what it searched for, which sources it used, whether those sources actually support the claims made, how reliable those sources are, or how old they are. The final output is a confident paragraph built on vibes.

Citation misattribution

The source exists and is real but the LLM misrepresents what it says, generalizes from a narrow finding, or blends multiple sources into a single unsourced claim.

Multi-agent cost failure

Long multi-agent deep research runs are expensive. When they fail halfway — wrong sources, reasoning drift, bad tool use — you've burned all the tokens and get nothing. Restart from scratch.

Product 1 — Open Source Tool

What it is

A free, open source search transparency layer. Shows every search query the LLM fires, maps each claim in the response to its exact source chunk, scores source reliability, and shows publication date. Provides a structured claim graph to the LLM before synthesis — replacing vibes-based amalgamation with structured evidence.

Pipeline

01Query— user input
↓
02Search— Tavily basic, 1 credit, returns URLs + snippets + date
↓
03Extract— Tavily /extract on top N URLs, clean markdown, ~0.2 credits/URL
↓
04Chunk— split pages into sentence/paragraph chunks
↓
05Claim Graph— extract atomic claims from chunks, score reliability, detect conflicts
↓
06Synthesize— LLM gets structured claim graph not raw text
↓
07Match— embed claims + chunks, cosine similarity, assign match score per claim
↓
08Display— studio UI with full transparency

Claim Graph Structure

Per claim, before LLM synthesis

  • Claim text (atomic, one fact)
  • Source URL + domain + date
  • Reliability score (domain heuristic: .gov/.edu/arxiv = high, vendor blog = low)
  • Number of independent sources supporting it
  • Conflicting claims from other sources flagged explicitly
  • Match score after synthesis (embedding similarity to source chunk)

Studio UI

Left panel

  • Final LLM response
  • Each claim highlighted inline
  • Click claim → jumps to source chunk
  • Color coded by match score

Right panel

  • Live feed of search queries fired
  • Source list with date + reliability badge
  • Exact matching chunk highlighted
  • Conflict warnings where sources disagree

Product 2 — Deep Research Engine (Paid)

What it is

A productized deep research tool built on top of the open source layer. Long-form multi-agent research runs with real-time steering. Open weight worker agents do the actual research. A steering agent watches each worker's chain-of-thought and intervenes mid-reasoning before errors compound. Closed model orchestrator plans the overall research strategy. Everything is traceable via the claim graph.

Architecture

Orchestrator (closed model)

  • Claude / GPT-4o for planning
  • Breaks research into subtopics
  • Spawns worker subagents per section
  • Final synthesis pass
  • Does not need to be steered

Worker agents (open weight)

  • Llama / Qwen / Mistral
  • Self-hosted inference
  • Full KV cache access
  • COT visible and injectable
  • Cheap to run many in parallel

Steering agent

  • Watches each worker's COT token by token as it streams
  • Skill file defines steering policy — no training needed, swappable per domain
  • Detects: low reliability source about to be cited, conflicting claims unacknowledged, reasoning drift from task, bad tool use pattern forming
  • Injects correction mid-COT before any action is taken
  • KV cache copy at decision points → branch, evaluate early tokens, pick best path, discard rest

Why deep research specifically

Long form = expensive failures

A 1-hour multi-agent research run on Claude burns $10–40 in API costs. If it fails halfway you restart from zero. Steering prevents the failure before it compounds.

Source quality matters most here

A wrong source cited in a research memo compounds across every downstream claim. The claim graph + steering catches this at the worker level before it reaches synthesis.

What the open source tool provides

The claim graph feeds directly into each worker subagent. Instead of each subagent synthesizing from raw search results independently, they receive structured evidence: claim → source → reliability score → conflict flags. Workers are no longer working on vibes. The steering agent uses the same graph to know exactly what to watch for.

Key Decisions

Search provider

Tavily basic search + extract as default. Pluggable so users can swap to Exa (better semantic quality) or Brave + Firecrawl (cheaper at scale). Users bring their own API key for the open source tool.

No /research endpoint

Tavily's /research costs up to 250 credits per call. We own the research loop ourselves — that's the product. Use only /search and /extract from Tavily.

Open weight workers, closed orchestrator

Orchestrator can be any closed model — no need to touch its internals. Workers are open weight so we own the inference, KV cache, and COT stream. Steering happens at the worker layer, invisible to the orchestrator.

Skill files not training

Steering agent behavior defined by swappable skill files. No retraining. Users can write their own steering rules. Different files for different research domains.
